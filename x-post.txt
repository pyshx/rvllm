rvLLM -- vLLM rewritten in Rust. One dev, 22 hours, $1,780.
10,291 tok/s. Beats Python vLLM up to N=256. 20x faster startup. 31x smaller. work in progress -- help me push this further.

what is it
rvLLM is a from-scratch Rust implementation of vLLM for serving LLMs on GPUs. drop-in OpenAI API replacement. same endpoints, same format, just point your client at the Rust server.

why
python vLLM transfers the entire logit matrix (74 MB for Qwen2.5-1.5B at batch 128) from GPU to host every decode step for sampling. rvLLM runs argmax directly on GPU and transfers 512 bytes. 150,000x reduction in PCIe traffic per step. no GIL, no GC, no PyTorch tensor metadata overhead. deterministic memory with Rust ownership. single 16 MB binary vs ~500 MB pip install.

architecture
- 23 Rust crates, strict dependency tree from GPU primitives to HTTP API
- 15 hand-written CUDA kernels compiled to PTX (sm_70 through sm_120)
- cuBLAS hgemm (FP16 tensor cores) for all linear projections
- f16 KV cache with paged block allocation (single cudaMalloc slab)
- FlashAttention-2 with paged KV cache and GQA support
- fused QKV projection (3 GEMMs -> 1), fused gate+up (2 GEMMs -> 1)
- GPU-side argmax kernel (greedy decode without logit transfer)
- CUDA graph capture/replay for decode steps
- mixed prefill+decode batching, chunked prefill
- continuous batching with preemption and prefix caching
- vectorized float4 loads across all kernels
- async HtoD metadata pipeline, packed PCIe transfers
- 10 model architectures (Llama, Mistral, Qwen2, Phi, Gemma, Mixtral, DeepSeek, GPT-NeoX, StableLM, Cohere)
- mock-GPU backend for development without NVIDIA hardware
- 793 tests across 23 crates

system comparison
- startup: 6s vs 121s (20x)
- binary: 16 MB vs ~500 MB (31x)
- CPU RSS: 348 MB vs 1,033 MB (3x)
- zero errors across 14,620 verified requests
- runs on 40GB cards where vLLM needs 73GB

GPU support
V100, T4, A100, RTX 3090/4090, H100, H200, B100, B200, RTX PRO 6000 Blackwell (sm_120). verified on A100 + B200 + RTX PRO 6000. CUDA 12.x and 13.x.

optimization phases
phase 1 (FP32): 3,191 tok/s -- baseline
phase 2 (FP16 hgemm + f16 KV cache): 8,339 tok/s -- 2.6x
phase 3 (fused GEMMs, CUDA graph fix, vectorized kernels, pre-alloc buffers): 10,291 tok/s -- 3.2x total
